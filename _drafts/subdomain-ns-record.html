When a domain is registered with one provider (e.g., GoDaddy), but its DNS is managed by another provider (e.g., AWS Route 53), here's how it works:

Key Components

  1. Domain Registrar (GoDaddy):

    • The registrar is where the domain name is registered and renewed.

    • It holds the authoritative record of your domain in the public WHOIS database.

    • It allows you to set the Name Server (NS) records for your domain, pointing to the DNS manager.

  2. DNS Manager (AWS Route 53):

    • The DNS manager is responsible for storing and serving your domain's DNS records (e.g., A, CNAME, MX, TXT, NS).

    • When the domain's name servers are set to AWS Route 53, AWS takes over DNS resolution for that domain.

An NS (Name Server) record specifies the authoritative name servers for a domain or subdomain. When adding an NS record for a subdomain, you're essentially delegating control of that subdomain to another set of name servers.
